Support Information Collector

Should errors occur in the context of Eclipse-based SAP development tools for which you, as an SAP customer, require more detailed support from SAP, it is of considerable importance that you provide all the required data for such a support case in order to speed up the troubleshooting process. For this purpose, SAP provides the Support Information Collector as a comfortable tool solution. With a few simple mouse clicks, you can generate a support information file. This file contains all the relevant data for your IDE or GUI version, as well as basic information on technical system environment data. You can then add this support file to your OSS problem message as an attachment.

In rare cases, however, it can happen that for detailed troubleshooting the SAP support team will request additional information from you. Then you can use the Support Information Collector utility to generate an extended support file version. This extended version can contain additional information about the RFC communication between the IDE client and the ABAP back-end system, or even heap dump, in the case of possible memory consumption issues.

Confidential Information

The Support Information Collector may collect confidential information. This includes the following:

  • Information that is available in Java system properties, such as command line parameters
  • Confidential data that has been written to the Eclipse log file
  • Confidential data stored in Eclipse preferences

    NOTE: Passwords stored in the Eclipse secure storage are not included.

  • When you create a heap dump, any data currently stored in memory is included. This may also enclose cached passwords.

RECOMMENDATION

If you think certain sensitive data should be excluded from the generated support files, make sure to review the created ZIP file before sending it to SAP support. Most data is stored in plain text format so that you can edit it to remove sensitive information.
